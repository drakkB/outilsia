Deux formats de sortie

Rapport PDF local ou rapport partageable : même décision, deux usages.

PDF modèleÀ garder localement : score, machine, modèle conseillé, preuve benchmark, commandes Ollama.
PDF upgradeÀ comparer avant achat : VRAM/RAM bloquantes, modèle débloqué, prix indicatif, piège à éviter.
Lien `/r/…`À partager volontairement : résumé anonyme de la config, benchmark et CTA scanner.
MemoryForgeÀ conserver dans Obsidian : machine, modèles, benchmarks, décisions et prochaines actions.

Laptop 16 Go sans GPU

CPU mobile · 16 Go RAM · 0 Go VRAM · Ollama optionnel

34/100

Verdict : utile pour découvrir, trop limité pour un assistant local moderne confortable.

Modèles à testerPhi-3 mini, Gemma 2B/3 1B, Qwen 0.5B/1.5B, petits modèles quantifiés CPU.
À éviterQwen 14B/32B, Llama 70B, image IA lourde, gros contextes et multi-agent.
Blocage principalPas de VRAM dédiée. Le CPU peut répondre, mais lentement et avec peu de marge.
Upgrade utileMini PC 32 Go pour labo léger, ou RTX 3060 12 Go si l'objectif est Ollama quotidien.

Rapport type : “ne pas acheter de SSD ou CPU avant d'avoir décidé si vous voulez vraiment une machine GPU.”

Le format que les bêta testeurs doivent nous aider à améliorer

1. ScoreCompréhensible en 5 secondes : excellent, correct, limité ou bloqué.
2. ModèlesCompatibles, limites, nouveaux modèles à tester et commandes Ollama.
3. BlocageVRAM, RAM, stockage, runtime, drivers ou refroidissement.
4. Achat utileUne recommandation matérielle seulement si elle débloque quelque chose.
